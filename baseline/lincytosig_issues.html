Core question: Can cell-type-specific signatures (LinCytoSig) outperform global signatures (CytoSig) for cytokine activity inference?

Current answer: No, at the donor level. CytoSig’s global signature outperforms all LinCytoSig variants in 4 of 4 single-cell datasets (median ρ advantage +0.03 to +0.14). Gene filtering helps (+12% to +102% improvement), but does not close the gap. The “best” cell-type selection introduces circularity that inflates apparent performance. See issues below for details.

Issue 1: Best-Selection Circularity OPEN

Problem

The “best LinCytoSig” variants select the top-performing cell type for each cytokine using bulk RNA-seq correlation (GTEx, TCGA, or combined), then validate on single-cell data. While bulk→SC is not the same metric, the selection criterion (Spearman ρ vs. target gene expression) is identical in both settings. This creates a semi-circular design: selection optimizes for the same measurement being used for validation.

Four Selection Strategies

We evaluated 4 selection strategies for choosing the best cell type per cytokine:

  1. Combined (GTEx+TCGA) — maximize average bulk ρ across both bulk datasets
  2. Combined + gene filter — same, but restricted to CytoSig’s 4,881 genes
  3. GTEx-only — maximize GTEx bulk ρ
  4. TCGA-only — maximize TCGA bulk ρ

Agreement Analysis

Of 44 cytokines, only 24 (55%) have the same best cell type across all 4 selection strategies. The remaining 17 cytokines (3 have limited data) select different cell types depending on the selection dataset.

Key finding: Nearly half (45%) of cytokines receive different “best” cell-type assignments depending on which bulk dataset is used for selection. This means the selection is not identifying a stable biological property but rather dataset-specific noise.

Issue 2: Case Studies — Where LinCytoSig Wins PARTIAL

Three case studies illustrate when cell-type-specific signatures outperform global CytoSig, and when selection artifacts distort the comparison. All correlation values are Spearman ρ at donor-only level.

Dataset composition caveat: CIMA and Inflammation Atlas Main are PBMC/blood-derived datasets — their cell types are exclusively immune cells (B, T, NK, monocytes, DCs, etc.). They contain no fibroblasts, epithelial cells, hepatocytes, or other stromal/tissue-resident cell types. When a non-immune LinCytoSig variant (e.g., Fibroblast__TGFB1, Keratinocyte__IL1B) is evaluated in CIMA or Inflammation Main, the signature trained on that cell type is being scored against immune cell pseudobulk — a biologically mismatched comparison. Only scAtlas Normal and scAtlas Cancer contain tissue-resident cell types. Values in the CIMA and Inflam Main columns for non-immune variants should be interpreted with this caveat.

Issue 3: Does the “Right” Cell Type Beat Global at Celltype Level? OPEN

The Problem with Donor-Level Comparison

Current comparisons (Section 5) evaluate all methods at donor-only level: pseudobulk across all cells per donor, regardless of cell type. This is inherently unfair to LinCytoSig because:

  1. PBMC paradox: Donor-level pseudobulk is a mixture of all cell types. CytoSig, trained on diverse cell types, is effectively a “mixture-level” signature. LinCytoSig, restricted to one cell type, is mismatched.
  2. Dilution: A Macrophage-specific signal gets diluted when averaged with fibroblasts, epithelial cells, etc. at the donor level.

The Right Test

The fair comparison for LinCytoSig is at the donor × celltype level: does Fibroblast__TGFB1 beat CytoSig TGFB1 specifically on fibroblast pseudobulk? The scatlas_normal_correlations.csv contains 47,586 LinCytoSig rows at celltype level with “matched” cell types.

This suggests that the specific cell type matters enormously, but the current selection strategy does not identify the right one.

A systematic celltype-matched evaluation across all 178 LinCytoSig signatures is needed to determine whether the right match can outperform CytoSig.

Issue 4: Gene Space Effect PARTIAL

Problem

LinCytoSig uses all ~20,000 genes, while CytoSig uses a curated set of 4,881 genes. Restricting LinCytoSig to CytoSig’s gene space (the “gene-filtered” variant) dramatically changes performance, suggesting many of the extra genes introduce noise rather than signal.

Gene Filter Effect on Median ρ

Median Spearman ρ across 20 matched cytokines, LinCytoSig matched cell-type signatures at donor level:

CIMA 0.082 0.166 0.225 0.334 +102%
Inflammation Main 0.133 0.201 0.285 0.330 +51%
scAtlas Normal 0.173 0.193 0.216 0.390 +12%
scAtlas Cancer 0.172 0.147 0.323 0.479 −15%

Source: method_comparison_8way_all.json, 20 matched cytokines per dataset. Filter effect = (filt − orig) / |orig| × 100%.

Key findings:

  • Gene filtering helps in 3 of 4 datasets, with dramatic improvement in CIMA (+102%) and Inflammation Main (+51%).
  • Even after filtering, LinCytoSig still underperforms CytoSig in all 4 datasets (gap of 0.023 to 0.176 in median ρ).
  • scAtlas Cancer is the exception: gene filtering slightly hurts performance (−15%), suggesting some of the extra genes carry useful cancer-specific signal.
  • SecAct dominates with the highest median ρ in all 4 datasets (0.330–0.479), despite using an entirely different signal source (spatial correlation).

Open Question

Which specific genes drive the noise? A gene-by-gene analysis of weights in the LinCytoSig signature matrix could identify “noise genes” — those with large absolute weights that do not contribute to the target-gene correlation. This would inform a more principled gene filtering approach than simply intersecting with CytoSig’s gene set.

Issue 5: Sample Size and PBMC Paradox DOCUMENTED

Small Sample Sizes

Many LinCytoSig cell types are derived from very few CytoSig experiments. The signature matrix is built from median log2FC of stimulation experiments, and cell types with fewer than 5 experiments produce noisy, unreliable signatures.

Examples of cell types with limited data:

  • Basophil: Featured in IL3 selection, but likely has very few experiments across all cytokines
  • ESC (Embryonic Stem Cell): Selected for BDNF, BMP4, NTF4 — limited relevance to adult tissue validation
  • Chondrocyte: Selected as “best” for IL1B (combined) — not present in any single-cell validation dataset

PBMC Paradox

CytoSig’s global signature is trained on median log2FC across all cell types in the database, including many PBMC experiments. At the donor level, pseudobulk is effectively a mixture of all cell types — structurally similar to PBMC. This means:

  1. CytoSig is already optimized for mixture-level signals because its training data spans diverse cell types, naturally averaging across cell-type-specific effects.
  2. LinCytoSig’s cell-type restriction hurts at donor level because it captures a narrower response that gets diluted in the mixture.
  3. PBMC__IFNG consistently underperforms despite PBMC being the “mixture-level” cell type: the PBMC-specific signature captures PBMC-specific effects (lymphocyte-dominated), not the broader tissue response.

Resolution: The comparison is structurally biased toward CytoSig at the donor level. LinCytoSig should be evaluated at the celltype level (Issue 3) for its intended use case. At the donor level, CytoSig is the correct choice.

Issue 6: Cross-Validation Design Gap OPEN

Current Design

The current validation pipeline uses:

  • Selection: Best cell type selected by bulk RNA-seq correlation (GTEx, TCGA, or combined)
  • Validation: Performance measured by Spearman ρ on single-cell pseudobulk (CIMA, Inflammation Main, scAtlas)

While bulk→SC validation is not fully circular (different data modality, different samples), both stages use the same metric (Spearman ρ between predicted activity and target gene expression). This means:

  1. The selection criterion rewards the same statistical pattern being measured in validation
  2. Overfitting to bulk-specific artifacts could still inflate validation performance
  3. No held-out test set exists for unbiased performance estimation

Proposed Solutions

Approach Description Pros Cons
Leave-one-atlas-out Select best on 3 SC datasets, validate on held-out 4th Unbiased; uses existing data Reduces training data; 4 is already small
GTEx-train / TCGA-test Select using GTEx only, validate on TCGA (or vice versa) Bulk-only; no SC needed for selection Still bulk→bulk; different confounders but same modality
Stability-based selection Require agreement across 2+ selection strategies Filters unstable selections; reduces circularity Eliminates 45% of cytokines with disagreement (Issue 1)
External validation Use parse_10M perturbation data as ground truth True independent validation Different experimental design; limited to 90 cytokines × 12 donors

Recommendation: Implement stability-based selection as a first step: only use “best” cell type assignments where all 4 selection strategies agree (24/44 = 55% of cytokines). For the remaining 20 cytokines, use the matched cell-type (LinCytoSig) or fall back to CytoSig (global). This eliminates the most circular cases without requiring new pipeline runs.
